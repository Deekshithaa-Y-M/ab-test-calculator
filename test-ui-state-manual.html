UI State Management Test Instructions

Task 8: Implement UI state management

This test verifies that:

  • ✓ clearResults function exists and resets results section
  • ✓ Input change event listeners clear results when inputs are modified
  • ✓ Results only display after calculate button is clicked

Test Procedure:

  1. Initial State Test:
    Expected: Results section should be hidden on page load
  2. Calculate Results Test:
    • Enter Control: Visitors = 1000, Conversions = 100
    • Enter Variant: Visitors = 1000, Conversions = 150
    • Click "Calculate Significance" button
    Expected: Results section should appear with conversion rates and confidence level
  3. Input Change Clears Results Test:
    • Change Control Visitors to 2000
    Expected: Results section should immediately disappear/hide
  4. Recalculate Test:
    • Click "Calculate Significance" button again
    Expected: Results section should reappear with updated values
  5. Multiple Input Changes Test:
    • Change Control Conversions
    • Change Variant Visitors
    • Change Variant Conversions
    Expected: Results should remain hidden after each change
  6. Final Calculation Test:
    • Click "Calculate Significance" button
    Expected: Results section should appear with new calculated values

Success Criteria:

  • ✓ Results section is hidden on initial page load
  • ✓ Results appear only after clicking Calculate button
  • ✓ Any input change immediately hides the results section
  • ✓ Results can be recalculated after input changes
  • ✓ No errors in browser console
